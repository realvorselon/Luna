# Theme Direction Boundary

This is a concise public-safe boundary note for Luna’s finished theme direction. This run does not change `prototype.html`, add code, add behavior, add controls, add persistence, or change the sample project.

Luna’s finished theme should be closer to **moonlit garden, quiet desk, and gentle return path**.

## Theme Layers

- **Moonlit:** the emotional atmosphere. Calm, soft, nighttime, safe to return.
- **Quiet desk:** the useful project workspace. Practical, readable, not just decorative.
- **Gentle return path:** the product experience. One clear way back into an unfinished project.
- **Garden / cultivation:** the action language. Orient, Grow, Prune, Rest, tend, leave alone, let wait.
- **Philosophy Garden:** an optional reflective layer, not the main engine.

## What Luna Should Not Become

Luna should not become:

- a kitchen / soup themed app
- a purely garden-themed app where every project must be treated as a plant
- a childish cozy toy
- a corporate productivity dashboard with cute words
- Jira with a flower crown
- a metaphor stew where moon, garden, desk, lantern, path, soup, and other images all compete equally

## Sample Project Boundary

The recipe-note / soup-note sample worked as a neutral public-safe fictional sample for testing the one-project return loop, but it is not the intended finished theme.

The default sample project may later stay recipe-based, or shift to something more Luna-adjacent, but that decision should wait. Do not change the sample project in this run.

## Future Polish Filter

Future visual and copy polish should use this direction as a filter:

- calm moonlit atmosphere
- useful desk-like clarity
- path-like return flow
- restrained garden/cultivation language
- no forced metaphor on every project
- no bloat
- no childishness
- no corporate task pressure

Preserve the Luna loop as the core: one project state, current mode, current goal, what changed last, one Next Action, Ignore For Now, Record the Change, and Rest.
