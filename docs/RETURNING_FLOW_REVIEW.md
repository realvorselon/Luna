# Returning Flow Review

This is a concise public-safe review note from the perspective of someone returning to an unfinished small project. It does not add code, prototype behavior, controls, persistence, or broader app planning.

Luna's product north star is to make returning easier.

The current prototype already shows the main returning bones: one project state, current mode, current goal, what changed last, one next action, what to ignore for now, record the change, and rest.

The saved project name and current goal help the prototype remember the project identity and current intention in the same browser. The clear-local-edits control gives a gentle way to return to the public-safe sample values.

The returning flow is visible, but still mostly static. Before adding more behavior, the next useful design question is whether the visible path feels clear enough for a person who has been away from a project.

Possible later improvements should be wording-first and small: clarify what to read first, make the return path feel more obvious, or make the one-project loop easier to scan.

Do not add new behavior yet. Do not add more persistence yet. Do not add more editable cards yet.

Do not add dashboard expansion, mode workflow, multi-project system, backend, accounts, database, sync, sessionStorage, cookies, analytics, app architecture, pricing, deployment planning, LLM implementation, or new Philosophy Garden content.

Preserve the dashboard ↔ prototype reversible path and “Return to dashboard” link.
