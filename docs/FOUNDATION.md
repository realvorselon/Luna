# Foundation Map

Luna's foundation docs are a small public map for returning to one small project, not a new app plan or a Philosophy Garden dashboard plan. They help a new visitor understand what is already here, where the current state lives, what changed last, what to do next, what to ignore for now, and how to keep each change calm and aligned.

## Start Here

- `README.md` is the front door. It explains Luna's current return path and points people toward the next places to read.
- `RULES.md` protects Luna's one-small-change rhythm and the expectation that related public records stay aligned.
- `STATE.md` records Luna's current identity, current goal, latest completed run, next suggested step, and notes.
- `CHANGELOG.md` records the public history of completed runs.
- `docs/EXAMPLE_RETURN.md` shows Luna in motion: return to one small project, see what changed last, choose or skip one next action, record the change, and rest.

## Guidance Notes

- `docs/STEWARDSHIP.md` describes how Luna may later choose a calm project mode and one next action, then record the work and rest.
- `docs/USER_GUIDANCE.md` explains how people can approach Luna without needing to be good at prompting.
- `docs/VISUAL_IDENTITY.md` explains how Luna should feel visually and in interface voice.
- `docs/APP_SEED.md` and `docs/MINIMAL_APP_SHAPE.md` keep the possible future app shape small and separate from building.

## Public Dashboard Files

- `project.json` stores the public dashboard content, including the current state, last completed run, current goal, and next action.
- `index.html` renders that content as the current public dashboard page.
- The Philosophy Garden content remains a gentle supporting reflection layer around the return path, not Luna's main engine.
