# First Usable Prototype Slice

This note defines Luna's first tiny usable prototype slice before it is built. It is a boundary note, not an architecture plan, feature plan, deployment plan, or visual redesign.

## Shape of the Slice

The first usable prototype should be a tiny lantern for returning to one project. It should stay local and simple first, and it should carry the same Luna return loop already shown by the self-demo dashboard.

The slice is only about one project. A person should later be able to enter or edit one small project state, then use that state to move calmly through the loop: return, orient, see what changed, choose one next action, see what can wait, record one change, and rest.

## Essential Project State

The first usable version only needs enough state to help someone return without turning Luna into a task manager:

- project name
- current status
- Current Mode
- current goal
- last completed run
- one Next Action
- Ignore For Now
- Record the Change
- rest / next step

## Loop It Should Protect

The first slice should help the person move calmly through this order:

1. Return to the project.
2. Orient around the current project state.
3. See what changed last.
4. Choose one next action.
5. See what can wait.
6. Record one change.
7. Rest before adding more.

## What This Slice Is Not

This note does not start building the prototype. It does not add editable UI, forms, local storage, backend, accounts, database, authentication, pricing, deployment planning, app architecture, LLM implementation, a multi-project system, dashboard expansion, visual redesign, images, icons, animations, or new Philosophy Garden content.

The first slice should feel like a small light left on for one project, not a productivity command center, backlog, or task manager.
