# Example Return Walkthrough

This is a concrete example of Luna helping one person return to one small project. It is fictional, public-safe, and intentionally small.

## Example Project

**Person:** Mina  
**Project:** A tiny browser game called **Moon Moth**, where a moth catches soft dots of light on one screen.

Mina has not opened the project for three weeks. She remembers liking the idea, but not where she stopped.

## Returning

When Mina comes back, Luna does not ask her to plan the whole game. Luna shows the saved project state in plain language:

- **Project:** Moon Moth, a tiny browser game about guiding a moth toward dots of light.
- **Current state:** One simple play screen exists. The moth can move left and right. The dots of light are only placeholders.
- **Current mode:** **Orient**.
- **What changed last:** Mina changed the moth from a square into a simple wing shape so the game felt more like its idea.
- **One suggested next action:** Make one dot of light disappear when the moth touches it.
- **Do not worry about yet:** score, sound, levels, menus, saving progress, better art, or making the game feel finished.

Luna keeps the return narrow: remember the project, notice the mode, choose one possible next action, and avoid the larger pile of ideas.

## One Small Return Step

Mina decides the suggested action is small enough for today. She makes one dot disappear when the moth touches it.

If Mina had skipped the action, Luna would still treat the return as useful. The record could simply say that Mina returned, reviewed the state, and chose to rest without changing the game.

## Recording What Changed

After the small step, Luna records the change in short public-safe language:

> The moth can now collect one dot of light. The next work should rest before adding score, sound, levels, menus, or more art.

The record does not need a long explanation. It only needs enough detail for Mina to return next time without starting over.

## Resting Before Adding More

Luna stops after the recorded change. The project is not asked to become a full game today.

The next state is simple:

- **Current mode:** **Rest**.
- **Last change:** The moth can collect one dot of light.
- **Next suggested step:** Rest before adding another mechanic, screen, style pass, or plan.

## What this demonstrates

This example shows Luna's core shape in motion:

- **One small project:** Moon Moth is one tiny browser game, not a collection of goals.
- **Saved project state:** Mina can see what exists without reconstructing it from memory.
- **Current mode:** Orient helps her return before doing more.
- **One next action:** Luna suggests one small possible step.
- **Short history:** The last change explains where the project stopped.
- **Ignore-for-now guidance:** Luna names the tempting things that can wait.
- **Record:** The completed step is written down plainly.
- **Rest:** Luna pauses before adding more features or plans.
