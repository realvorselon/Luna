# Visual Identity Note

Luna should feel calm, warm, and easy to return to. As a small public dashboard prototype that may later become a calm companion for small projects, her future visual direction should support the project's purpose: helping people return to a project, remember where they left off, choose one safe next step, and avoid overwhelm.

This note is guidance only. It is not a request to add code, CSS, images, icons, a dashboard section, a data object, a roadmap, or a larger product plan.

## Desired Visual Qualities

- A soft dashboard personality that feels like a quiet place to return to a small project.
- Clear, readable cards that make the current goal, last completed run, and Next Action easy to find.
- Gentle colors that feel warm without becoming distracting.
- Calm spacing that gives each idea enough room to breathe.
- Subtle mode badges for Orient, Grow, Align, Prune, Reflect, and Rest if Luna later needs mode labels.
- A lantern-like focus on the Next Action, so the safest next step is easy to notice.
- Visual warmth without clutter.

## What to Avoid

- Busy gamification.
- Mascot overload.
- Too many animations.
- Dramatic or stressful colors.
- Decoration that makes the next step harder to find.
- Turning Luna into a giant productivity command center.

## Voice and Personality in the Interface

Luna can sound gentle and encouraging, but the wording should stay practical and useful. Warmth should help the user act, not bury the interface in poetry.

Interface text should make it easier to return, choose, and continue with one safe next step. If a phrase feels lovely but makes the action less clear, choose the clearer phrase.

## Rest Before Visual Polish

Before adding any visual identity feature or CSS polish, Luna should rest or review this note. Future visual work should stay public-safe, beginner-friendly, and limited to one small useful change at a time.
